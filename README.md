# yt-grab

A bash script that downloads YouTube videos as MP4 or extracts the audio as MP3. Pass video IDs instead of full URLs. Missing dependencies install themselves on Arch, Debian/Ubuntu, and Fedora.

## Features

- Download a single video, multiple videos, or an entire playlist
- MP4 (best video + audio, muxed) or MP3 (highest VBR audio extract)
- Auto-detects the OS and installs `yt-dlp` and `ffmpeg` if they aren't present
- Output folder defaults to `~/yt-downloads` and is created on first run
- Accepts both bare video IDs and full URLs

## Requirements

- bash 4 or newer
- `sudo` access for the first-run dependency install (skipped entirely if `yt-dlp` and `ffmpeg` are already on `$PATH`)

If your distro isn't supported by the auto-installer, install `yt-dlp` and `ffmpeg` manually from your package manager and the script picks them up on next run.

## Supported distros (auto-install)

| Family | Detected for | Package manager |
| --- | --- | --- |
| Arch | Arch, CachyOS, EndeavourOS, Manjaro, Garuda, Artix | `pacman` |
| Debian | Debian, Ubuntu, Pop!_OS, Linux Mint, elementary, Zorin | `apt` |
| Fedora | Fedora, Nobara, Bazzite | `dnf` (uses `ffmpeg-free` from default repos) |

Detection reads `ID` and then `ID_LIKE` from `/etc/os-release`, so derivatives not on the explicit list usually still resolve correctly.

## Installation

```bash
# Grab the script
curl -O https://raw.githubusercontent.com/tempusthales/yt-grab/main/yt-grab.sh

# Make it executable
chmod +x yt-grab.sh

# Optional: drop it on your PATH
sudo mv yt-grab.sh /usr/local/bin/yt-grab
```

First run prompts to install `yt-dlp` and `ffmpeg` if either is missing. Use `-y` to skip the prompt for non-interactive use.

## Usage

```text
yt-grab -f <mp4|mp3> [-o <dir>] [-l <playlist_id>] [-y] [<video_id> ...]
```

| Flag | Description |
| --- | --- |
| `-f FORMAT` | Output format: `mp4` or `mp3` (required) |
| `-o DIR` | Output directory (default: `~/yt-downloads`) |
| `-l ID` | YouTube playlist ID (downloads the whole playlist on its own, or anchors playback when combined with a video ID) |
| `-y` | Assume yes to dependency install prompt |
| `-h`, `/?` | Show help |

### Examples

Single video as MP4:

```bash
yt-grab -f mp4 cGDX6cbqzsk
```

Single video as MP3:

```bash
yt-grab -f mp3 cGDX6cbqzsk
```

Multiple videos in one run:

```bash
yt-grab -f mp3 cGDX6cbqzsk dQw4w9WgXcQ
```

Whole playlist as MP4:

```bash
yt-grab -f mp4 -l PLxxxxxxxxxx
```

Whole playlist as MP3:

```bash
yt-grab -f mp3 -l RDcGDX6cbqzsk
```

Custom output directory:

```bash
yt-grab -f mp3 -o ~/Music/podcasts cGDX6cbqzsk
```

Non-interactive (skip the install prompt):

```bash
yt-grab -f mp3 -y cGDX6cbqzsk
```

### Help and bare invocation

```bash
yt-grab           # shows help
yt-grab /?        # shows help (Windows-style)
yt-grab -h        # shows help
```

## Finding video and playlist IDs

For a video URL like `https://www.youtube.com/watch?v=cGDX6cbqzsk`, the ID is the part after `v=`: `cGDX6cbqzsk`.

For a playlist URL like `https://www.youtube.com/playlist?list=PLxxxxxxxxxx`, the ID is the part after `list=`: `PLxxxxxxxxxx`.

Common playlist ID prefixes:

- `PL...` user-curated playlist
- `RD...` autogenerated radio/mix
- `UU...` user uploads
- `OLAK5uy_...` YouTube Music auto-album

Full URLs are also accepted, so paste whichever form you have on hand.

| Code | Meaning |
| --- | --- |
| 0 | Success, or help shown |
| 1 | Validation error, install aborted, or unsupported OS |
| Other | Passed through from yt-dlp |

## Troubleshooting

**"OS could not be identified"**: your distro isn't on the supported list and `ID_LIKE` in `/etc/os-release` doesn't match either. Install `yt-dlp` and `ffmpeg` manually with your package manager and re-run.

**"command not found" after install completes**: uncommon, but can happen if the package install succeeded but the binary went to a directory not on `$PATH`. Check `$PATH` against the install location.

**yt-dlp errors during extraction**: YouTube changes its internals often, and yt-dlp ships fixes within days. Update first:

```bash
# Arch
sudo pacman -Syu yt-dlp

# Debian/Ubuntu
sudo apt update && sudo apt install --only-upgrade yt-dlp

# Fedora
sudo dnf upgrade yt-dlp
```

If your distro repo is behind upstream, install the current release via `pipx`:

```bash
pipx install yt-dlp
```

**Fedora codec issues**: the auto-installer uses `ffmpeg-free` from default Fedora repos, which handles MP4 muxing and MP3 encoding for typical use. If you hit a codec error on something exotic, enable RPM Fusion and swap to the full build:

```bash
sudo dnf swap ffmpeg-free ffmpeg --allowerasing
```
